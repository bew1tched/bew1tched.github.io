Aufgabenstellung

Prüfen eines mit Hilfe der Lösung von ESA 2 erstellten Grammatik mit einem Kellerautomat.

Technische Umsetzung

Das Layout wurde grundsätzlich mit HTML5 und CSS3 erstellt.
Es handelt sich um eine Grid-View, in dem die einzelnen Bereiche aufgeteilt sind.
Das Design der Buttons und Textfelder wurde mit Bootstrap generiert.
Sämtliche Funktionalität wurde mit JavaScript implementiert.

Bei dem Graph handel es sich um ein SVG, das mit Adobe Illustrator erstellt wurde.

Fachliche Umsetzung

Der Automat prüft Zeichen für Zeicchen die eingegebene Grammtik.
Als Startwert wird immer eine Ziffer oder eine "(" erwartet. Ist dies nicht der Fall, wird false zurück gegeben.
Bei Zeichen, die nicht an der ersten Stelle befinden, wird zusätzlich der Vorgänger überprüft.

Geöffnete Klammern werden mit 'X' in den Keller geschrieben. Wird eine Klammer geschlossen, wird ein 'X' aus dem Keller geholt.

Beispiel geschlossene Klammer:

Es wird in Zeile 312 folgendes überprüft:

if (isClosed(current_char) && (isClosed(last_char) || isNumber(last_char)))

Der Vorgänger einer geschlossenen Klammer kann nur eine weitere geschlossene Klammer oder aber eine Ziffer sein, ist dies nicht der Fall, wird der Ausdruck false.
Ähnlich wie in diesem Beispiel wird auch bei allen anderen Schritten vorgegangen.
Die möglichen Übergänge können der Tabelle entnommen werden.
Zusätzlich werden die Klammern mitgezählt. Wird eine Klammer geöffnet, wird diese in den Keller geschrieben, wird nun später wieder eine Klammer geschlossen, wird eine Klammer aus dem Keller entfernt. Am Ende muss der Keller genau leer sein. Es kann also weder ein 'X' im Keller verbleiben noch darf versucht werden ein 'X' aus dem Keller zu holen, obwohl dieser bereits leer ist.
